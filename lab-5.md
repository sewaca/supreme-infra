# Список выявленных дефектов

## BUG-1: Кнопка «Заказать» активируется без выбора пункта выдачи при выключенном режиме PDF

| Атрибут             | Описание                                                                                                                                    |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| ID дефекта          | BUG-1                                                                                                                                       |
| Связанный тест-кейс | 2.4, 4.2                                                                                                                                    |
| Серьёзность         | Major                                                                                                                                       |
| Приоритет           | High                                                                                                                                        |
| Краткое описание    | При выключенной опции «Только виртуальная справка (PDF)» форма позволяет активировать кнопку «Заказать» без выбора пункта получения справки |

**Шаги к воспроизведению**

1. Открыть страницу `/profile/references`.
2. Выбрать любой валидный тип справки, например `По месту работы родителей`.
3. Убедиться, что чекбокс «Только виртуальная справка (PDF)» выключен.
4. Не выбирать значение в поле «Где получить справку».
5. Проверить состояние кнопки «Заказать».

**Фактический результат**

Кнопка «Заказать» становится активной, несмотря на то, что обязательное поле «Где получить справку» не заполнено.

**Ожидаемый результат**

Кнопка «Заказать» должна оставаться неактивной до выбора пункта получения справки, если режим PDF не включён.

## BUG-2: При вводе отсутствующего типа справки не появляется пользовательская опция

| Атрибут             | Описание                                                                                                               |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| ID дефекта          | BUG-2                                                                                                                  |
| Связанный тест-кейс | 5.3                                                                                                                    |
| Серьёзность         | Major                                                                                                                  |
| Приоритет           | Medium                                                                                                                 |
| Краткое описание    | При вводе значения, отсутствующего в списке типов справок, форма не добавляет пользовательский вариант как новую опцию |

**Шаги к воспроизведению**

1. Открыть страницу `/profile/references`.
2. Перейти в поле «Тип справки».
3. Ввести значение `zzz`, отсутствующее в списке предложенных типов.
4. Открыть выпадающий список вариантов.

**Фактический результат**

В списке не появляется пользовательская опция `zzz`, выбрать нестандартный тип невозможно.

**Ожидаемый результат**

Если среди типов справок нет подходящего варианта, в выпадающем списке должна отображаться пользовательская опция с введённым значением, чтобы можно было заказать нестандартную справку.

# Выводы

В ходе лабораторной работы было выполнено ручное функциональное тестирование формы заказа справок в системе diploma.sewaca.ru. Всего было проверено 28 тест-кейсов, из которых успешно прошли 26, не прошли — 2.

В процессе тестирования были выявлены следующие дефекты:

- BUG-1 (Major): кнопка «Заказать» активируется без выбора пункта выдачи при выключенном режиме PDF;
- BUG-2 (Major): поле «Тип справки» не позволяет выбрать пользовательский вариант, если подходящий тип отсутствует в списке.

Большинство сценариев работы формы реализованы корректно: выбор типа справки, работа чекбокса виртуальной справки, поведение поля получения и базовая логика блокировки/активации кнопки в стандартных случаях функционируют ожидаемо. Однако выявленные дефекты затрагивают важные пользовательские сценарии: оформление бумажной справки и заказ нестандартного типа справки.

Рекомендуется устранить дефект BUG-1 в первую очередь, поскольку он приводит к отправке некорректно заполненной формы. После этого следует исправить BUG-2, чтобы восстановить заявленную поддержку пользовательского ввода нестандартных типов справок.
