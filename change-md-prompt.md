## 1. Описание функционала

### Аутентификация и аккаунт (`core-auth`, `web-auth-ssr`)

- Регистрация и вход, сессии, валидация сессии, выход из сессий.
- Восстановление пароля (запрос, проверка токена, сброс).
- Двухфакторная аутентификация: challenge, проверка и верификация.
- Токены для CalDAV: выдача, отзыв, внутренняя валидация.
- Внутренние API для смены email и пароля пользователя (для интеграции с другими сервисами).

### Профиль и персональные данные (`core-client-info`, `web-profile-ssr`)

- Профиль пользователя, персональные данные, группы, поиск пользователей при регистрации, пакетная выборка пользователей.
- Настройки аккаунта (чтение/обновление), смена email и пароля.
- Рейтинг и геймификация: достижения, оценки, улучшения оценок, уровень, рейтинги, статистика, серии (streak).
- Приоритеты по предметам: список выбора, сохранение приоритетов, чтение приоритетов пользователя.

### Заявки, справки, общежитие, заказы (`core-applications`)

- Список заявлений пользователя, уведомления по заявкам, инициализация пользователя (внутренний вызов).
- Общежитие: подача заявки, согласие родителя.
- Заказы: список, детали, PDF, счётчики.
- Справки: список, детали, заказ справки, отмена, продление хранения, PDF.

### Расписание (`core-schedule`, `web-schedule-ssr`)

- Расписание и экзамены для групп и преподавателей, шаблоны расписания.
- Экспорт в CalDAV (ICS) для групп и преподавателей по токену.
- Админ-часть: аудитории, семестры, шаблоны (в т.ч. массовые операции), события сессий, переопределения расписания, синхронизация преподавателей.

### Сообщения (`core-messages`, `web-messages-ssr`)

- Диалоги: список, создание прямого диалога, чтение/отправка сообщений, пометка прочитанным, удаление диалога и сообщений, обновления (polling/long-poll по маршруту).
- Рассылки (broadcast): создание, список, группы для рассылок.
- Поиск сообщений и пользователей, профили пользователей для мессенджера.
- Загрузка файлов к сообщениям; WebSocket для real-time (маршрут `ws`).
- SSR-страницы: список переписок, чат по id, новое сообщение, поиск, рассылки и создание рассылки.

### Документы (`web-documents-ssr`)

- Просмотр/представление документов: зачётная книжка, студенческий билет (страницы под авторизованного пользователя).

### Файлы (`system-files-storage`)

- Загрузка файлов в объектное хранилище (MinIO), генерация превью (миниатюр) — на уровне API сервиса.

### Инфраструктура репозитория (не прикладной UI, но часть «проекта»)

- Сборка Docker-образов, релизы с canary и ручным промоутом, Helm-чарты, генерация роутеров и overrides, секреты и БД PostgreSQL для сервисов.

## 2. Описание страниц (пользовательские маршруты Next.js)

Маршруты разнесены по нескольким Next-приложениям; в проде они обычно склеиваются за ingress/reverse-proxy. Уровень `auth_level: valid` означает ожидание авторизованной сессии.

### `web-auth-ssr`

| Путь | Назначение |
|------|------------|
| `/login` | Вход. |
| `/register` | Регистрация. |
| `/forgot-password` | Восстановление пароля. |
| `/web-auth-ssr/*` | Служебные пути сервиса. |

### `web-profile-ssr`

| Путь | Назначение |
|------|------------|
| `/profile` | Профиль (корень раздела). |
| `/profile/data` | Персональные / учётные данные. |
| `/profile/dormitory` | Общежитие. |
| `/profile/orders` | Заказы. |
| `/profile/rating` | Рейтинг. |
| `/profile/references` | Справки. |
| `/profile/scholarship`, `/profile/scolarship` | Стипендия (в т.ч. опечатка в пути для совместимости). |
| `/profile/settings` | Настройки. |
| `/profile/settings/change-email` | Смена email. |
| `/profile/settings/change-password` | Смена пароля. |
| `/profile/subjects-ranking` | Ранжирование / приоритеты предметов. |
| `/web-profile-ssr/*` | Служебные пути сервиса. |

### `web-schedule-ssr`

| Путь | Назначение |
|------|------------|
| `/schedule` | Расписание (общий вход в раздел). |
| `/schedule/group` | Расписание группы. |
| `/schedule/teacher` | Расписание преподавателя. |
| `/web-schedule-ssr/*` | Служебные пути сервиса. |

Дополнительно у сервиса есть API-роуты `GET /api/schedule`, `POST /api/caldav-setup` (не «страницы», а эндпоинты приложения).

### `web-messages-ssr`

| Путь | Назначение |
|------|------------|
| `/messages` | Список переписок. |
| `/messages/new` | Новое сообщение / новый диалог. |
| `/messages/search` | Поиск. |
| `/messages/broadcast` | Рассылки. |
| `/messages/broadcast/new` | Создание рассылки. |
| `/messages/[id]` | Переписка по идентификатору. |
| `/web-messages-ssr/*` | Служебные пути сервиса. |

### `web-documents-ssr`

| Путь | Назначение |
|------|------------|
| `/documents/gradebook` | Зачётная книжка. |
| `/documents/student-id-card` | Студенческий билет. |
| `/web-documents-ssr/*` | Служебные пути сервиса. |

---

*Маршруты синхронизируются с кодом через генератор (`pnpm run generate:router`); при расхождении с `router.yaml` в сервисах источником правды является дерево `app/` в соответствующем Next-проекте.*
