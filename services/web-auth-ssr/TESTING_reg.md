# Тестирование регистрации

Все пользователи ниже **существуют в core-client-info** (миграция `006_unregistered_users.sql`), но **не имеют аккаунта в core-auth** — их можно использовать для проверки нового флоу регистрации.

## Как тестировать

1. Открыть `/register`
2. Ввести **Фамилию** и **СНИЛС** из таблицы ниже
3. Ввести любой **новый email** (не должен уже быть в системе) и **пароль** (мин. 6 символов)
4. Нажать «Найти аккаунт» → система покажет карточку с данными
5. Нажать «Да, это я» → аккаунт создаётся, редирект на `/profile`
