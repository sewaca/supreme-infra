# Промпт: сервис расписания обучения (университет)

## Цель

Разработать сервис для **отображения и управления** расписанием обучения в университете.

## Стек и инфраструктура

- **Backend:** FastAPI.
- **Данные:** собственная БД, **PgBouncer**.
- **Инфраструктура** (папка сервиса, БД и т.п.) создаётся **генератором** — нужно опираться на него и использовать его при создании проекта.

## Сущности и внешние связи

Сервис хранит и связывает данные со **`core-client-info`**. Достаточно **договорённости** (например, хранение внешних ID и согласованные названия сущностей); детальный контракт API в промпте не требуется.

| Сущность в сервисе | Связь                                                         |
| ------------------ | ------------------------------------------------------------- |
| Учебные группы     | с группами в `core-client-info`                               |
| Преподаватели      | с пользователями в `core-client-info`                         |
| Предметы           | с предметами в `core-client-info`                             |
| Аудитории          | только в этом сервисе                                         |
| Расписание         | с учебными группами, преподавателями, предметами, аудиториями |

## Роли пользователей

- Администраторы
- Преподаватели
- Студенты

## Логика расписания

1. **Основное расписание** на протяжении **двух календарных недель** задаётся один раз как **постоянный шаблон** и **циклически повторяется** на всём семестре (какие дни недели и какие пары в какой «неделе цикла» — те же, пока шаблон не изменят).
2. В **отдельные дни или слоты** этот шаблон могут **заменять форс-мажорные изменения** (отмена, перенос, другая аудитория и т.п.); при отдаче данных из API по календарным периодам эти изменения **нужно учитывать**.
3. Как именно хранить шаблон и переопределения (в т.ч. в отдельных таблицах) — **детали реализации**, важна описанная семантика.

Имеет смысл явно различать:

- просмотры **базового шаблона семестра** (без учёта форс-мажоров) — см. отдельные пункты в функционале ниже;
- просмотры на календарный период (день / неделя / месяц) — **с учётом** форс-мажорных переопределений.

## Функциональность (API)

- Расписание **определённой группы** на день / неделю / месяц (фильтр по дате начала и конца периода).
- Расписание **зачётов и экзаменов** за семестр с фильтром по **группе**.
- Расписание **пар определённого преподавателя** на день / неделю / месяц (фильтр по дате начала и конца периода).
- Расписание **зачётов и экзаменов** за семестр с фильтром по **преподавателю**.
- Расписание **группы на семестр** — **базовый циклический шаблон**, **без** форс-мажорных изменений.
- Расписание **преподавателя на семестр** — то же (**базовый шаблон**), **без** форс-мажорных изменений.

## Технические заметки

- **Пагинация:** на усмотрение исполнителя (страничная или курсорная), с упором на **производительность** и **нагрузку на БД**.
- **Тип пары** — строка. Допустимые значения: лекция, практическое занятие, лабораторная работа, консультация, экзамен, зачёт, семинар, факультатив.
- **Номер пары** (слот в расписании дня) **может указываться** для любых занятий, включая зачёт, экзамен и консультацию в сессии.
