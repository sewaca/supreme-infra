Привет!

Мне нужно чтобы ты разработал сервис для отображения и управления расписанием обучения в университете
Сервис на фреймворке fast-api, со своей БД и PGBouncer
Инфра (папка сериса, бд, и т.п.) целиком создается генератором, используй его

Информация которую будет хранить сервис: учебные группы (должны быть связаны с группами в core-client-info), преподаватели (должны быть связаны с пользователями в core-client-info), предметы (должны быть связаны с предметами в core-client-info), аудитории, расписание (должно быть связанно с учебными группами, преподавателями, предметами, аудиториями)
Пользователи сервиса: администраторы, преподаватели, студенты

Расписание на семестр – хранится в отдельной табличке. Это расписание заданное на две недели, которое циклно повторяется в течении семестра. Дополнительно, поверх него, могут быть какие-либо форс-мажорные изменения, которые надо будет учитывать в результате работы ручки

Примерный функционал:

- посмотреть расписание определенной группы на день / неделю / месяц (фильтр по дате старта / конца периода)
- посмотреть расписание зачетов и экзаменов за семестр с фильтром по группе
- посмотреть расписание пар определенного преподавателя на день / неделю / месяц (фильтр по дате старта / конца периода)
- посмотреть расписание зачетов и экзаменов за семестр с фильтром по преподавателю
- посмотреть расписание группы на семестр (должно храниться в отдельной табличке, это расписание без форс-мажорных изменений)
- посмотреть расписание преподавателя на семестр (должно храниться в отдельной табличке, это расписание без форс-мажорных изменений)

Пагинацию можешь реализовать на свое усмотрение: страничную или курсорную, главное чтобы был высокий перформанс приложения и база не страдала
тип пары можно хранить в виде строчки. могут быть такие типы: лекция, практическое занятие, лабораторная работа, консультация, экзамен, зачет, семинар, факультатив
