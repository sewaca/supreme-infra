# Промпт: фронтенд-интеграция CalDAV-токенов (core-schedule)

## Контекст

В сервисе `core-schedule` реализован iCal-фид для синхронизации расписания с внешними
календарями (Google Calendar, Apple Calendar, Outlook).

Фид доступен по URL:

```
GET /core-schedule/caldav/{token}/groups/{group_name}/calendar.ics
GET /core-schedule/caldav/{token}/teachers/{teacher_id}/calendar.ics
```

Токен — непрозрачная строка (urlsafe base64, 43 символа), хранится в таблице `caldav_token`
в БД сервиса. Управление токенами через API:

| Метод    | Путь                  | Auth | Описание                                              |
| -------- | --------------------- | ---- | ----------------------------------------------------- |
| `POST`   | `/caldav/tokens`      | JWT  | Создать токен. Возвращает значение токена (один раз!) |
| `GET`    | `/caldav/tokens`      | JWT  | Список токенов текущего пользователя (без значения)   |
| `DELETE` | `/caldav/tokens/{id}` | JWT  | Отозвать токен (owner-check, 403 если чужой)          |

### Схема ответов

**POST /caldav/tokens** — тело запроса:

```json
{ "device_name": "CalDav calendar" }
```

Ответ (`201`):

```json
{
  "id": "uuid",
  "token": "abc123...",
  "device_name": "CalDav calendar",
  "created_at": "2026-03-27T21:00:00Z"
}
```

> ⚠️ `token` возвращается **только при создании** — сохранить сразу.

**GET /caldav/tokens** — ответ:

```json
[
  {
    "id": "uuid",
    "device_name": "CalDav calendar",
    "created_at": "2026-03-27T21:00:00Z",
    "revoked_at": null
  }
]
```

**DELETE /caldav/tokens/{id}** — `204 No Content`.

## Задача

Реализовать UI для управления CalDAV-токенами в личном кабинете пользователя и показать
инструкцию по подписке на календарь.

Найди в фронтенде страницу профиля / настроек пользователя и добавь туда раздел
**"Синхронизация с календарём"**.

---

## Требования к UI

### Раздел "Синхронизация с календарём"

Раздел содержит:

1. **Список активных токенов** — загружается при открытии страницы через `GET /caldav/tokens`.
   Показывать только неотозванные (`revoked_at === null`).
   Каждая строка: название устройства + дата создания + кнопка **Отозвать**.

2. **Кнопка "Добавить календарь"** — открывает модальное окно / inline-форму.

### Модальное окно "Добавить календарь"

Шаг 1 — **выбор субъекта**:

- Radio/select: **Группа** или **Преподаватель**
- Если Группа: поле ввода названия группы (например, `ИКПИ-25`)
- Если Преподаватель: поле выбора UUID преподавателя (или поиск по имени через существующий API)

Шаг 2 — **создание токена**:

- `POST /caldav/tokens` с `device_name = "CalDav calendar"`
- Показать сгенерированный URL для подписки:
  ```
  https://diploma.sewaca.ru/core-schedule/caldav/{token}/groups/{group_name}/calendar.ics
  ```
  или
  ```
  https://diploma.sewaca.ru/core-schedule/caldav/{token}/teachers/{teacher_id}/calendar.ics
  ```
- Кнопка **Копировать URL**

Шаг 3 — **инструкция** (показывается сразу после копирования / по кнопке "Как добавить?"):
Показать инструкцию (см. ниже) под скопированной ссылкой. Можно accordion/details.

### Отзыв токена

При нажатии **Отозвать**:

- `DELETE /caldav/tokens/{id}`
- Убрать строку из списка
- Показать toast: "Доступ к календарю отозван"

## Инструкция для пользователя (вставить в UI)

Вставь эту инструкцию в интерфейс дословно (можно в accordion "Как добавить в календарь?"):

---

### Как добавить расписание в календарь

Скопируй ссылку выше и следуй инструкции для своего устройства.

**iPhone / iPad (Календарь)**

1. Открой **Настройки → Приложения → Календарь → Учётные записи**
2. Нажми **Добавить учётную запись → Другое → Добавить подписной календарь**
3. Вставь скопированную ссылку → **Далее → Сохранить**
4. Расписание появится в приложении «Календарь» и будет обновляться автоматически

**Google Calendar (браузер)**

1. Открой [calendar.google.com](https://calendar.google.com)
2. Слева нажми **+** рядом с «Другие календари» → **По URL**
3. Вставь ссылку → **Добавить календарь**
4. Календарь появится в списке и будет синхронизироваться (обычно раз в сутки)

**Outlook (настольное приложение)**

1. Перейди в **Календарь → Добавить календарь → Из Интернета**
2. Вставь ссылку → **OK**

**Отмена синхронизации** — нажми **Отозвать** напротив нужного устройства в списке.
После отзыва ссылка перестанет работать; уже добавленный календарь можно удалить вручную.

## Технические детали для реализации

- Все запросы к `core-schedule` идут через общий API-клиент (`@supreme-int/api-client`).
  Нужно добавить методы для CalDAV-токенов (или использовать прямой fetch если клиент не обновлён).
- `token` из ответа `POST /caldav/tokens` нигде не сохранять в state после закрытия модалки —
  пользователь уже скопировал URL.
- При ошибке `403` на отзыве (чужой токен) — показать "Нет доступа".
- При ошибке `409` (уже отозван) — убрать строку из списка без toast.
- `device_name` пока фиксированный `"CalDav calendar"` — не показывать поле в UI.
